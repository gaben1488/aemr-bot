# DESIGN.md — дизайн-система документации aemr-bot

> Единый источник стиля для всех страниц-документов. Решения приняты,
> не «на выбор». Anti-slop, доступность прежде всего.

## Сцена (одно предложение, форсирует решения)

Пожилой житель Камчатки с телефоном в руке и уставший муниципальный
оператор за старым монитором открывают справку при дневном свете и хотят
быстро, без страха ошибиться, понять, что делать.

Форсирует: **светлая тема** (дневной свет, пожилые глаза), **крупный
шрифт**, **высокий контраст**, **спокойствие**, **много воздуха**,
минимум украшений, никакого тёмного «техно».

## Color strategy: Restrained

Тёплая бумага + чернила + один основной акцент (хвойно-зелёный) +
тёплый второй акцент (охра) очень дозированно для «человеческого» тепла
на страницах жителя. Никакого гос-синего (первый рефлекс) и
медицинской бирюзы (второй рефлекс).

Палитра в OKLCH (никаких чистых #000/#fff, все нейтрали тонированы тёплым):

```css
:root {
  /* Поверхности — тёплая бумага */
  --paper:        oklch(0.985 0.004 85);   /* фон страницы */
  --paper-raised: oklch(0.995 0.003 85);   /* приподнятые блоки */
  --paper-sunken: oklch(0.955 0.006 80);   /* утопленные блоки, код */

  /* Чернила — тёплый почти-чёрный, не #000 */
  --ink:          oklch(0.255 0.012 70);   /* основной текст */
  --ink-soft:     oklch(0.435 0.012 70);   /* вторичный текст */
  --ink-faint:    oklch(0.60 0.010 70);    /* подписи, мета */

  /* Основной акцент — глубокий хвойный (доверие, природа Камчатки) */
  --pine:         oklch(0.46 0.062 168);
  --pine-deep:    oklch(0.38 0.060 168);   /* ховер, нажатие */
  --pine-tint:    oklch(0.95 0.022 168);   /* мягкая заливка-плашка */
  --pine-line:    oklch(0.86 0.030 168);   /* тонкие границы акцента */

  /* Тёплый второй акцент — охра (человеческое тепло, дозированно) */
  --ochre:        oklch(0.70 0.090 72);
  --ochre-tint:   oklch(0.955 0.030 78);

  /* Состояния (доступный контраст на бумаге) */
  --danger:       oklch(0.52 0.150 27);    /* запреты, мошенники */
  --danger-tint:  oklch(0.955 0.030 27);
  --warn:         oklch(0.60 0.110 65);
  --ok:           oklch(0.52 0.090 150);

  --line:         oklch(0.90 0.006 80);    /* обычные границы */
}
```

Тёмная тема: поддержать `prefers-color-scheme: dark` мягко (бумага →
тёплый графит oklch(0.20 0.008 70), чернила светлеют), но светлая —
первичная. Не делать неон на чёрном.

## Typography

**Без внешних шрифтов (CDN).** Гос-сервис, страницы должны открываться
офлайн и мгновенно. Системный стек, аккуратно настроенный.

```css
--font-display: Georgia, "Iowan Old Style", "Times New Roman", serif;
--font-text: system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", sans-serif;
--font-mono: ui-monospace, "SF Mono", "Cascadia Code", Consolas, monospace;
```

- Заголовки — **сериф** (Georgia): редакционный, тёплый, внушает доверие,
  отлично рендерит кириллицу везде.
- Текст — **гуманистический санс** (system-ui): максимальная читаемость
  для пожилых.
- Базовый размер крупный: **18px** на телефоне, **19–20px** на широком.
  Длина строки 62–70ch. Межстрочный 1.65 для текста.
- Иерархия контрастом масштаба и веса, шаг ≥1.25. Шкала (rem от 18px):
  h1 2.6 / h2 1.9 / h3 1.4 / h4 1.15 / body 1 / small 0.86.

## Layout

- Редакционная одноколоночная читаемая ширина (~66ch) по центру.
- На широком экране — «липкое» оглавление сбоку (навигация по разделам),
  на телефоне оглавление сворачивается наверх.
- Много вертикального воздуха, ритм переменный (не одинаковые отступы).
- **Карточки — крайний случай.** Роли на главной — НЕ одинаковая сетка
  плиток с иконками (это слоп), а крупные текстовые «двери» с разной
  длиной описания, разделённые линиями.
- Вложенных карточек не бывает. Контейнером оборачиваем не всё подряд.

## Компоненты

- **Плашка-врезка** (callout): фон-тинт + полная тонкая граница + ведущий
  значок/слово. НИКАКИХ боковых цветных полос (`border-left` как акцент
  запрещён). Виды: подсказка (pine-tint), осторожно (ochre-tint),
  опасно/мошенники (danger-tint).
- **Шаги** (нумерованный сценарий): крупная цифра-ведущий слева, текст
  справа, без рамок.
- **Команда/кнопка бота**: моноширинный в утопленной плашке (paper-sunken),
  рядом — что делает простыми словами.
- **Таблица**: для сравнения по осям (роли × права, ключи настроек).
  Тёплые границы, зебра очень мягкая, заголовок плотнее.

## Motion

- Только ease-out (expo/quart), коротко (120–220мс). Без bounce/elastic.
- Не анимировать layout-свойства. Только opacity/transform.
- Уважать `prefers-reduced-motion: reduce` — выключать.
- Якорный переход к разделам — плавный, но отключаемый.

## Доступность (приоритет)

- Контраст ≥ WCAG AA, для текста стремимся к AAA.
- Семантический HTML, `lang="ru"`, скип-ссылка к содержанию.
- Цели нажатия ≥ 44px. Читается при увеличении 200%.
- Видимый фокус на ссылках/кнопках (pine-обводка).
- Печать: чистый чёрный текст на белом, навигация скрыта (`@media print`).
- Эмодзи — только как функциональные значки в самом продукте; в вёрстке
  документации значки рисуем inline-SVG или текстом, без декоративных
  эмодзи в заголовках.

## Anti-slop чек (обязательный перед сдачей страницы)

Если глядя на страницу можно без сомнений сказать «это сделал ИИ» —
переделать. Конкретные запреты: одинаковые карточки-плитки, эмодзи в
заголовках, градиентный текст, стеклянные блоки по умолчанию, большая
цифра-метрика, боковые цветные полосы, em-dash, «в современном мире».
