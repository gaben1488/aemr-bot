# Администрация ЕМР. Обратная связь

Чат-бот в мессенджере MAX для приёма обратной связи от жителей Елизовского муниципального района. Админ-панель — служебный групповой чат в MAX, куда падают обращения и где операторы отвечают реплаями.

Образец UX — бот «Солодов. Обратная связь» (правительство Камчатского края).

## Стек

- **Python 3.12** + библиотека `maxapi`
- **PostgreSQL 16** через SQLAlchemy 2.x + Alembic
- **APScheduler** в том же процессе (ежесуточная статистика, бэкап)
- **openpyxl** для выгрузки XLSX по команде из админ-группы
- **Docker Compose** — два контейнера: `bot` и `db`
- **Long polling** в режиме доставки событий (без webhook → без HTTPS-эндпойнта на MVP)

Никакого FastAPI, Nginx, Redis, React, мини-приложения — на MVP не нужны.

## Главное меню жителя — три кнопки

1. **Написать обращение** — после согласия на обработку ПДн (один раз) и нативного запроса контакта (имя + телефон) житель пишет свободным текстом + опционально фото и геолокация.
2. **Мои обращения** — список обращений жителя со статусами и последними ответами оператора.
3. **Контакты** — подменю: электронная приёмная (ссылка), приём граждан (расписание + телефон), экстренные службы (clipboard-кнопки), расписание автобусов УДТХ (ссылка).

## Админ-группа в MAX

Создаётся служебная группа, куда добавлены бот и операторы (координатор АЕМР, ИТ, при необходимости — отдельные группы под ЕГП). Поведение бота:

- При новом обращении бот шлёт в группу карточку: номер `#NNN`, имя, телефон, текст обращения, фото и геолокация (если есть).
- Оператор отвечает **реплаем** на это сообщение — бот переправляет ответ жителю в личный диалог.
- В группу падает подтверждение: «✉️ Ответ успешно отправлен пользователю. Обращение #NNN помечено как завершённое».
- Команды в группе: `/stats today|week|month` (XLSX-выгрузка), `/reopen NNN` (вернуть обращение в работу), `/close NNN` (закрыть без ответа).

## Уровень защищённости персональных данных

УЗ-4 по ПП №1119: иные ПДн, третий тип угроз, менее 100 000 субъектов, не сотрудники. Аттестация провайдера ФСТЭК не требуется. Подробнее — `docs/ADR-001-architecture.md`, раздел «Безопасность».

## С чего начать

1. Зарегистрировать тестового бота через `@MasterBot` в MAX, получить токен.
2. Создать две служебные группы в MAX: для приёма обращений и для ЕГП-команды.
3. Купить виртуалку (рекомендация — Aeza CLT-2: 2 vCPU / 4 ГБ / 60 ГБ NVMe, ~1 700 ₽/мес).
4. Запустить скелет кода (создаётся следующим шагом, после ревью документов).
