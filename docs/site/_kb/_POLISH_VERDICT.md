# ЧЕСТНЫЙ ВЕРДИКТ: нужна ли ещё полировка

> Свод трёх линз (impeccable-audit / стандарты / мета-вывод) по проекту aemr-bot
> и единой офлайн-вике `docs/site/index.html`. Сверено руками с живым деревом
> 2026-06-02. Подход MLP self-host: enterprise-overkill осознанно отсекается.

## Вердикт одной фразой

**Код и вика готовы как артефакт (артефакт 9/10, код 82.8% / разнесён / security-чисто),
но боевой сбор ПДн заблокирован тремя НЕ-кодовыми P0 — текст согласия, Постановление
№2129 как первоисточник, реквизиты оператора ПДн; всё остальное — улучшающая полировка (P1)
или сознательный MLP-отказ.**

## Главная развилка трёх линз — и как она разрешена

Линзы согласны по фактам, расходились по тяжести одного пункта — отсутствие `<h1>`.
Стандарты ставили P0 («формальное AA-нарушение, ловит любой аудит»), impeccable — P2.
Честный разбор: **в WCAG нет критерия, который буквально требует `<h1>`.** Иерархия,
стартующая с `<h2>`, — валидный HTML5 и сама по себе не проваливает 1.3.1. Это реальная
деградация точки входа для скринридера и дешёвый фикс, но **не юридический AA-hard-fail.**
Поэтому `<h1>` — **P1, не P0.** Настоящие P0 — это юридический gate из мета-линзы, который
кодом не закрывается и реально блокирует законность сбора ПДн.

## P0 — реально сделать до полного спокойствия

Все три P0 — **НЕ КОД** (юрист / owner). Это gate законности, не полировка.

1. **Текст согласия `seed/consent.md`** (733 байта, не тронут с 13 мая). Те же 3 предложения:
   политика «по ссылке, которую укажет администратор» = ссылка в никуда; перечень ПДн =
   только «имя и номер телефона», хотя собираются адрес/текст/вложения/гео/max_user_id.
   Под ч.4 ст.9 152-ФЗ — оспоримое согласие. Правка seed-текста (без редеплоя) + юрист.
   **Самый острый остаток.**
2. **Постановление №2129 как первоисточник.** Plain-text акта в репозитории нет. Несущая
   опора «59-ФЗ не применяется» держится на доверии. Организационный + юрист.
3. **Реквизиты оператора ПДн + ответственный по ст.18.1 + утверждённая Политика + `policy_url`.**
   Поля пусты. До заполнения сбор ПДн нелегитимен.

> Вывод по P0: проект **НЕ готов «как есть» к боевому сбору ПДн** — но строго по
> юридическим, не техническим причинам. Как технический артефакт — готов.

## P1 — улучшит (genuine, не overkill)

**Вика (a11y / адаптив / перф):**
- **10 mermaid-диаграмм без текст-альтернативы** (нет `<title>`/`<desc>`/`aria-label`, нет
  видимой подписи — класс `.diagram__cap` определён, но не использован). Слепой оператор
  получает безымянный SVG. WCAG 1.1.1 (A). Чинится в шаблоне сборки разом. `/impeccable harden`.
- **Поиск молчит для скринридера при найденных результатах** — `role="status"` (`index.html:294`)
  висит только на «ничего не найдено». WCAG 4.1.3 (AA). `/impeccable harden`.
- **Нет `<h1>`** (подтверждено: 0 вхождений) — точка входа для NVDA теряется, SPA усугубляет.
  Повысить стартовый заголовок активной секции или sr-only глобальный `<h1>`. (Развилка выше: P1.)
- **Нет `@media (forced-colors: active)`** (подтверждено: 0) — российские слабовидящие в Windows
  High Contrast теряют тинты плашек/бейджей и `stroke=currentColor`-иконки. Прямое попадание в
  цель ГОСТ Р 52872-2019. `/impeccable adapt`.
- **Touch-таргеты** `.copy-btn` (~28px) и `.nav a` (~37px) ниже 44px на тач (AA 24px проходит,
  AAA 44px — нет). `/impeccable adapt`.
- **`lang`-разметка** связных англ. фрагментов в тексте (WCAG 3.1.2 AA); код в `<code>`/`<pre>`
  не трогать. Точечно.
- **Фокус не переносится при SPA-смене секции** (`show()`), скринридеру не анонсируется —
  после `show(id)` фокус на заголовок (`tabindex="-1"`+`.focus()`).

**Эксплуатация (genuine слепые зоны):**
- **Offsite-бэкап.** Код готов (`config.py BACKUP_S3_*`, `services/db_backup.py`), но выключен —
  бэкап в named-volume на той же VPS, потеря диска уносит и БД, и копии. Включить S3/rsync = конфиг.
- **Обязательный внешний dead-man-switch.** `HEALTHCHECK_URL` реализован, но опционален; сейчас
  единственный детектор полного отказа — жалоба жителя. Сделать обязательным + Healthchecks.io (free).
- **DR-учение end-to-end + bus-factor=1** как явный accepted-risk + runbook передачи. Организационный.
- **5 doc-пробелов в `_kb2/`** (вика пересобрана Jun 2 ПОСЛЕ аудита, часть болей влита — `ФСТЭК`,
  `УЗ-4`, `docker system prune`, `утечк` есть; пробелы остались): runbook диска (`df -h`/`du`),
  restore НА БОЕВУЮ базу, плейбук утечки ПДн (ст.21, 24/72ч), матрица мер ФСТЭК №21 для УЗ-4 +
  «календарь ИБ», decommission-runbook. Дёшево: текст + пересборка `_assemble.py`.

**Код (единственная genuine кодовая правка):**
- **SLA в рабочих часах.** `stats.py:136-137` считает `answered_at - created_at` без вычета
  ночей/выходных; `calendar_ru.py` есть, `work_time.py` отсутствует. Регламент «4 рабочих часа»
  → отчётность завышает просрочки, по метрике судят операторов. Решение спроектировано, не внедрено.

## P2 / НЕ НАДО — MLP, overkill (сознательный отказ)

**P2 по настроению (косметика):**
- E-4: 13 анонимных `async def _(` в `start.py` — в трейсбэке прода виден только `_`. Наблюдаемость.
- D-2 dead-код остаток (`send_to`, `is_whitelisted_url`, `reset_cache_for_tests`, `has_defangable_url`,
  `reset_all`) — `ruff`/`vulture` шумят. Косметика.
- Поиск перестраивает DOM по всему документу на каждый ввод (дебаунс 140мс смягчает) — `/impeccable
  optimize` ТОЛЬКО если на реальном офисном железе видны фризы, иначе пропустить.
- Опциональная кнопка сброса поиска (× для `type=search`). P3.
- 3 противоречия БЗ↔коду (статус «муниципальный документ», `/forget` флаг vs DELETE, «300 vs 1024») —
  проверить, что не воскресли в `_kb2/`; правка документальная.

**НЕ ДЕЛАТЬ (overkill для MLP self-host long-polling бота):**
- Mutation-тесты (mutmut/cosmic-ray) — enterprise-практика, ROI отрицательный при 80% + характеризации.
- Покрытие >80% — гейт уже 80 (план перевыполнен), погоня к 90%+ = diminishing returns; polling-loop
  корректнее под `# pragma: no cover`.
- Полное слияние трёх диспетчеров в один — уже есть практичный минимум (`route_for` + payload-builders);
  полное слияние = высокий риск по сердцу роутинга ради эстетики.
- Массовая lazy-import чистка (109 function-local) — многие настоящие cycle-breakers; трогать точечно.
- Продуктовые ходы (пульс района, пост-ответный опрос, воронка-инструментирование) — стратегия, не
  тех-долг; out-of-scope для MLP-запуска.
- JSON-LD / публичный `llms.txt` — артефакт офлайновый self-host, классический сценарий неприменим;
  максимум опциональная Markdown-выгрузка `docs/site/kb.md` для локальной LLM оператора.
- Масштаб УЗ-3 / отдельный воркер рассылки 70k — «триггеры на будущее», зафиксировать пороги
  (`count(users)>80000`), строить сейчас = overkill.

## Сухой остаток (порядок)

1. **P0 (юрист/owner, gate):** consent.md → Постановление №2129 → реквизиты+ответственный+`policy_url`.
   Без них сбор ПДн нелегитимен. Это не полировка.
2. **P1 дёшево-и-сейчас (конфиг, не разработка):** включить offsite-S3 + обязательный `HEALTHCHECK_URL`
   (код готов) + добить 5 doc-пробелов в `_kb2/` и пересобрать.
3. **P1 a11y вики одним проходом** `/impeccable harden` → `/impeccable adapt`: alt диаграмм + `.diagram__cap`,
   анонс поиска, `<h1>`, forced-colors, touch-таргеты, lang, фокус при SPA-смене.
4. **P1 код:** подключить working-hours SLA (`stats.py:137`, добавить `work_time.py`).
5. **P1 организационно:** одно DR-учение + bus-factor как явный accepted-risk.
6. **НЕ делать:** mutation-тесты, покрытие>80, слияние диспетчеров, продуктовые ходы, lazy-import чистка,
   JSON-LD/публичный llms.txt, масштаб УЗ-3.
