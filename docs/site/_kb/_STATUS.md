## P0 — gate законности сбора ПДн (НЕ КОД: юрист / owner)

1. **Текст согласия `seed/consent.md`** — самый острый остаток. Те же 3 предложения с 13 мая:
   политика «по ссылке, которую укажет администратор» = ссылка в никуда; перечень ПДн = только
   «имя и номер телефона», хотя собираются адрес/тема/текст/вложения/гео/max_user_id. Под ч.4 ст.9
   152-ФЗ — оспоримое согласие.
   **Точный технический root-cause (сверено 2026-06-04):** seed/consent.md ещё и *технически
   мёртв* — SCHEMA требует подстроку `{policy_url}` (settings_store.py:463 `required_substr`),
   а seed её НЕ содержит → `get_consent_request_text` (settings_store.py:623) при отсутствии
   плейсхолдера отдаёт **hardcoded fallback**, игнорируя seed. Следствие: текущая правка
   seed/consent.md жителю не доходит вовсе.
   **Чинить:** переписать `seed/consent.md` с обязательным `{policy_url}` + полным перечнем ПДн,
   под правооснование v8 (п.2 ч.1 ст.6 152-ФЗ, без 59-ФЗ — Постановление №2129; `/forget` —
   действующий путь отзыва, существует в коде). Правка seed-текста идёт без редеплоя, но это
   юридический документ → утверждает owner/юрист, не агент. _(Это заместило ошибочный черновик
   `_proposals/consent_152fz_draft.md`, удалённый 2026-06-04: тот опирался на п.4+59-ФЗ против v8
   и ложно утверждал «/forget которой нет».)_
2. **Постановление №2129 как первоисточник.** Plain-text акта в репозитории нет. Несущая опора
   «59-ФЗ не применяется» держится на доверии. Организационный + юрист.
3. **Реквизиты оператора ПДн + ответственный по ст.18.1 + утверждённая Политика + `policy_url`.**
   Поля пусты. До заполнения сбор ПДн нелегитимен.

## P1 — улучшит (genuine, не overkill)

**Вика (a11y / адаптив / перф) — перепроверить после HTML 2.0:**
- mermaid-диаграммы без текст-альтернативы (`<title>`/`<desc>`/`aria-label`, видимая подпись
  `.diagram__cap`) — WCAG 1.1.1 (A). Чинится в шаблоне сборки разом.
- Поиск молчит для скринридера при найденных результатах — `role="status"` только на «не найдено»
  (`index.html`). WCAG 4.1.3 (AA).
- Нет `<h1>` — точка входа для NVDA теряется, SPA усугубляет (развилка линз: это P1, не P0 —
  в WCAG нет критерия, буквально требующего `<h1>`).
- Нет `@media (forced-colors: active)` — Windows High Contrast теряет тинты/иконки (ГОСТ Р 52872-2019).
- Touch-таргеты `.copy-btn` (~28px) и `.nav a` (~37px) ниже 44px (AAA).
- `lang`-разметка англ. фрагментов (WCAG 3.1.2 AA); код в `<code>`/`<pre>` не трогать.
- Фокус не переносится при SPA-смене секции — после `show(id)` фокус на заголовок.

**Эксплуатация (genuine слепые зоны):**
- **Offsite-бэкап** — код готов (`config.py BACKUP_S3_*`, `services/db_backup.py`), но выключен;
  бэкап в named-volume на той же VPS. Включить S3/rsync = конфиг.
- **Обязательный внешний dead-man-switch** — `HEALTHCHECK_URL` реализован, но опционален; единственный
  детектор полного отказа сейчас — жалоба жителя. Сделать обязательным + Healthchecks.io (free).
- **DR-учение end-to-end + bus-factor=1** как явный accepted-risk + runbook передачи. Организационный.
- **doc-пробелы в `_kb2/`** (источник — `_audit/_PLAN.md`): runbook диска, restore на боевую базу,
  плейбук утечки ПДн (24/72ч), матрица мер ФСТЭК №21 для УЗ-4, decommission-runbook. Текст + пересборка.

**Код (единственная genuine кодовая правка):**
- **SLA в рабочих часах.** `stats.py:136-137` считает `answered_at - created_at` без вычета
  ночей/выходных; `calendar_ru.py` есть, `work_time.py` отсутствует. Регламент «4 рабочих часа»
  → отчётность завышает просрочки. Решение спроектировано (см. `_LEDGER.md` Часть II, спецификация
  telegram_analytics, Этап A), не внедрено.

## P2 / НЕ НАДО — MLP, overkill (сознательный отказ)

**P2 (косметика):** анонимные `async def _(` в трейсбэке; остаток dead-кода (`ruff`/`vulture` шум);
поиск перестраивает DOM на каждый ввод (дебаунс 140мс смягчает — `optimize` только при реальных фризах);
кнопка сброса поиска.

**НЕ ДЕЛАТЬ (overkill для MLP self-host long-polling бота):** mutation-тесты; покрытие >80
(гейт уже выполнен); полное слияние трёх диспетчеров (риск по сердцу роутинга ради эстетики);
массовая lazy-import чистка (многие — настоящие cycle-breakers); продуктовые ходы (пульс района,
пост-ответный опрос) — стратегия, не тех-долг; JSON-LD / публичный `llms.txt` (офлайновый self-host);
масштаб УЗ-3 / отдельный воркер рассылки 70k — зафиксировать пороги (`count(users)>80000`), строить сейчас = overkill.
