# Документация проекта «ЧАТ-БОТ / АДМИНИСТРАЦИЯ ЕМО»

Чат-бот обратной связи Администрации Елизовского муниципального округа в мессенджере MAX (`@aemo_chat_bot`, id 274913354).

## Главное — единая вики

Вся пользовательская и инженерная документация собрана в **самодостаточную офлайн-вики**:

- **[site/index.html](site/index.html)** — один HTML-файл, открывается двойным кликом, работает без интернета. Разделы: Обзор · Жителю · Оператору · Администратору · Разработчику · Как устроено · Безопасность и ПДн · Правовое · Справочник · Журнал решений. Поиск по всей базе (Ctrl/Cmd+K) и интерактивные диаграммы (на весь экран, скачивание SVG/PNG).
- Источник вики: `site/_kb2/*.html` (фрагменты разделов) + `site/_kb/_shell.html` (каркас, CSS, JS); собирается скриптом `site/_kb/_assemble.py` в `index.html`. Синхронность стережёт CI-тест `bot/tests/test_wiki_in_sync.py` и pre-commit-хук. **`index.html` руками не правят** — правят фрагменты и пересобирают.

## Регламент (канон)

- **[Регламент_AEMR-bot_MAX_v8.md](Регламент_AEMR-bot_MAX_v8.md)** + **[.docx](Регламент_AEMR-bot_MAX_v8.docx)** — **действующий prod-регламент v8**, источник истины по процедурам, ролям, срокам и правовой рамке: сообщение через бот — «сообщение из открытого источника» по Постановлению № 2129 (не обращение по 59-ФЗ); основание обработки ПДн — п.2 ч.1 ст.6 152-ФЗ; 4 роли (coordinator/aemr/egp/it); целевые сроки ответа — Таблица 2; раздел 8 — порядок взаимодействия с профильными подразделениями.
- История сборки v8 (что из v7 / owner-правок / итераций вошло) — `site/_kb/_sources/_reglament_changelog.md`.
- `archive/Регламент.docx` — **устаревший** утверждённый v7, superseded by v8 (оставлен как исторический документ).

## Персональные данные

- **[Политика_конфиденциальности_ИС_Чат-бот_MAX.md](Политика_конфиденциальности_ИС_Чат-бот_MAX.md)** — «Политика конфиденциальности при обработке персональных данных в информационной системе "Чат-бот MAX"», расширенная политика обработки ПДн под правовую рамку v8 (источник для `PRIVACY.pdf` после утверждения распоряжением и заполнения юридических плейсхолдеров). `.docx`-версия не поддерживается — актуален только `.md`.
- [PRIVACY.pdf](PRIVACY.pdf) — PDF, который бот отправляет жителям по команде `/policy`.
- Текст согласия живёт в `seed/consent.md` (runtime), содержит `{policy_url}` и краткий состав/цели ПДн — синхронизирован с `bot/aemr_bot/copy/citizen_funnel.py::CONSENT_REQUEST` и `settings_store.DEFAULTS["consent_text"]`.

## Разработка

- [DEPS.md](DEPS.md) — дисциплина зависимостей (`uv.lock`, hash-pin, Dependabot, pip-audit). Архитектура, БД, миграции, тесты, особенности `maxapi` — в разделе «Разработчику» вики.

---

> **Внутренняя кухня вики** (`site/_kb/`): `_LEDGER.md` — выверенный реестр фактов (КОД = истина, супер-канон, от него «питаются» регламент и вика); `_STATUS.md` — текущий статус проекта; `_audit/_PLAN.md` — план doc-пробелов; `_sources/` — провенанс сборки; `_archive/` — свёрнутые процессные снимки (с индексом `_archive/README.md`). Прежние ролевые `.md` (PRD / RUNBOOK / SYSADMIN / SECURITY / DEVELOPER / HOW_IT_WORKS и т.п.) упразднены — их содержание перенесено в вики и в регламент v8.
