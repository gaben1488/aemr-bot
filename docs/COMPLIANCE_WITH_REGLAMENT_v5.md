# COMPLIANCE: соответствие Регламента v5 фактическому коду

> **Важно.** После согласования v5 в код добавлены фичи (PR #26, #27, #31, #32, #33, #34), которые формально расширяют функциональность Регламента, не противоречат ему. Дельта-правки для версии **v6** подготовлены в `docs/Регламент_v6_draft.md` — черновик согласовывается с автором Регламента и юристом, утверждается распоряжением Администрации. До утверждения v6 настоящая матрица отслеживает соответствие фактического кода **v5**, явно фиксируя расширения как `OK (extension — v6 draft)`.

**Метод:** построчная сверка каждого нумерованного пункта Регламента с реализацией в коде по состоянию на 21.05.2026 (после мержа PR #26, #27, #28, #29). Источник истины для кода — `bot/aemr_bot/`. Для документации — `docs/`.

**Статус:**
- **OK** — требование Регламента полностью реализовано в коде или зафиксировано в документации.
- **drift** — фактическое поведение отклоняется от формулировки Регламента; основание и решение указаны явно.
- **TODO** — требование не реализовано; нужна отдельная задача.

### Drift § 74 — `appointment_text` района/округа

**Факт:** В `bot/aemr_bot/services/settings_store.py` поле `appointment_text` в DEFAULTS содержит формулировку «Приём граждан временно исполняющим полномочия Главы Елизовского **муниципального района** А.С. Гончаровым…».

**Регламент v5 (§§ 1-2, 4 и др.)** использует «Елизовский муниципальный **округ**» как название организации.

**Происхождение:** Коммит `e8dadce53b "Update settings_store.py 123"` от 20.05.2026 заменил «округа» → «района» в этом конкретном default-строке. Пользователь (владелец репозитория) **подтвердил намеренность** этого изменения.

**Решение:** не фиксить. Это контролируется через UI «Настройки бота» / `/setting` — IT может изменить в любой момент без редеплоя. Текущее значение `appointment_text` в проде = то, что было выставлено владельцем; default в коде — только начальное значение для свежей установки.

**Рекомендация (некритическая):** при следующем пересмотре Регламента v5 либо привести default в коде к «округа» (если Регламент финализируется именно так), либо синхронизировать терминологию во всех текстах.
