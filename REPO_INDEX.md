# Индексация репозитория aemr-bot

Этот файл — навигационная точка для быстрой индексации проекта в ChatGPT, Claude, Cursor, NotebookLM и других инструментах анализа кода.

Полный индекс кода не хранится в Git, потому что он является производным локальным артефактом и может случайно раздуть репозиторий. Вместо этого в `scripts/make_repo_index.py` лежит безопасный генератор индекса.

## Быстрая команда

Из корня репозитория:

```bash
python scripts/make_repo_index.py
```

На выходе появится файл:

```text
aemr-bot-index.md
```

Его можно загружать в ИИ-инструменты как единый слепок репозитория.

## Только дерево файлов

Если нужен легкий обзор без содержимого файлов:

```bash
python scripts/make_repo_index.py --output aemr-bot-tree.md --tree-only
```

## Что индексируется

Генератор включает небольшие текстовые файлы исходников и конфигурации: Python, Markdown, TOML, YAML, SQL, JSON, Dockerfile, `.gitignore`, `.env.example` и похожие безопасные текстовые форматы.

Генератор исключает `.env`, секреты, токены, ключи, логи, базы данных, дампы, архивы, PDF, изображения, виртуальные окружения, кэши, сборочные артефакты и бинарные файлы.

## Настройка размера

По умолчанию отдельный текстовый файл пропускается, если он больше 160 KB. Лимит можно изменить:

```bash
python scripts/make_repo_index.py --max-file-kb 300
```

## Для локальной проверки перед загрузкой в ИИ

Перед отправкой индекса во внешний сервис стоит выполнить поиск по чувствительным словам:

```bash
rg "TOKEN|SECRET|PASSWORD|API_KEY|BOT_TOKEN|DATABASE_URL|PRIVATE_KEY|ACCESS_TOKEN" aemr-bot-index.md
```

Если `ripgrep` не установлен, на Windows его можно поставить так:

```powershell
winget install BurntSushi.ripgrep.MSVC
```

## Почему полный индекс не коммитится

Файл `aemr-bot-index.md` добавлен в `.gitignore`. Это намеренно: индекс должен быть локальным одноразовым артефактом, который пересоздается перед ревью и не смешивается с исходным кодом.
